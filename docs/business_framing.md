# 业务定位：贷前准入策略项目

## 一句话定位

本项目不是 Kaggle 刷榜项目，而是一个贷前信用风险准入策略项目：在贷款申请发生时，基于申请信息、外部征信历史、历史申请和还款行为，识别未来还款困难风险，并将风险评分转化为可解释、可执行、可监控的准入策略。

## 业务问题

金融机构在贷前环节面临三个目标的平衡：

1. 控制坏账风险：降低未来还款困难客户的通过率。
2. 保持业务规模：避免过度拒绝本来可以还款的好客户。
3. 控制运营成本：人工审核资源有限，需要把审核资源用于最值得复核的申请。

因此，本项目最终不只输出模型 AUC，而是输出策略：

- `approve`: 低风险，自动通过。
- `manual_review`: 中高风险，进入人工复核、补充材料、降额或加强验证。
- `decline`: 高风险，拒绝或强风控处理。

## 标签定义

Home Credit 的 `TARGET` 可视为“申请人在未来出现还款困难”的二分类标签：

- `TARGET = 0`: 未出现还款困难。
- `TARGET = 1`: 出现还款困难。

注意：这不是欺诈标签，也不是实时交易风险标签，而是贷后表现标签。因此项目应定位为贷前信用风险/准入策略，而不是交易反欺诈。

## 数据口径

建模粒度：一笔贷款申请，即 `SK_ID_CURR`。

申请时点：以 `application_train/test` 中当前申请为时点。

历史观察窗口：申请发生前已经存在的客户历史贷款、征信、还款、信用卡/POS表现。

表现窗口：当前申请之后用于确认是否还款困难的未来表现，只能用于生成标签，不能进入特征。

## 关键业务难点

1. 多源数据合并：主申请表、外部征信、历史申请、还款流水、信用卡余额、POS余额。
2. 粒度不一致：申请级、历史贷款级、月度状态级、还款分期级。
3. 审批时点可用性：必须确保所有特征在当前申请时已经可见。
4. 缺失值含义：缺少征信、收入、职业、历史记录本身可能是风险信号。
5. 样本不平衡：坏样本占比较低，不能只看 accuracy。
6. 策略代价：拒绝坏客户是收益，误拒好客户是机会成本。

## 评估指标

模型指标：

- AUC
- KS
- Lift
- Precision@K
- Recall@K

策略指标：

- 自动通过率
- 人工复核率
- 拒绝率
- 拒绝组坏样本捕获率
- 复核组坏样本命中率
- 自动通过组坏样本率
- 分群稳定性

解释与监控：

- SHAP 全局特征重要性
- 个体 reason code
- 特征 PSI
- 分数分布漂移
- 分群坏样本率稳定性

## 项目输出

- 数据源映射与 ER 关系说明。
- 原始数据质量报告。
- 防泄露审计清单。
- 申请级特征集市。
- 信用评分模型。
- 贷前准入策略表。
- SHAP reason code。
- PSI/score drift 监控框架。
- 面向简历和面试的项目总结。
