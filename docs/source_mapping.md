# 数据源映射

## application_train / application_test

粒度：当前贷款申请，一行对应一个 `SK_ID_CURR`。

用途：

- 建模主表。
- `application_train` 包含 `TARGET`，用于训练和验证。
- `application_test` 不包含 `TARGET`，用于模拟待审批申请。

业务含义：

- 当前申请金额、年金、收入、贷款用途、家庭状态、职业、居住信息、外部评分等。

注意：

- `TARGET` 只能作为标签，不能作为特征。
- `EXT_SOURCE_*` 是强预测变量，但需要解释为外部评分/外部信息源。

## bureau

粒度：外部征信历史贷款，一名客户可对应多条 `SK_ID_BUREAU`。

用途：

- 构建外部征信历史负债、逾期、贷款类型、账户状态、历史信用活跃度。

关键特征方向：

- 历史贷款数量。
- Active / Closed 账户数量。
- 逾期金额。
- 当前债务余额。
- 授信额度。
- 贷款类型分布。
- 最近一次外部贷款距离申请日多久。

## bureau_balance

粒度：外部征信历史贷款的月度状态。

用途：

- 描述每笔外部贷款的历史状态轨迹。

关键特征方向：

- 历史状态 `0/1/2/3/4/5/C/X` 的占比。
- 最近状态。
- 历史逾期月份数。
- 账户状态波动。

## previous_application

粒度：Home Credit 历史申请，一名客户可有多次历史申请。

用途：

- 构建客户在本机构历史申请行为与审批结果。

关键特征方向：

- 历史申请次数。
- 通过/拒绝/取消比例。
- 历史申请金额与当前申请金额差异。
- 历史产品类型。
- 最近一次历史申请时间。

## installments_payments

粒度：历史分期还款计划与实际还款。

用途：

- 构建历史还款纪律。

关键特征方向：

- 逾期天数。
- 提前还款/逾期还款次数。
- 实际还款金额与应还金额差异。
- 最近还款表现。

## credit_card_balance

粒度：历史信用卡月度余额。

用途：

- 构建额度使用、循环负债、逾期状态。

关键特征方向：

- 额度使用率。
- 余额/授信额度。
- ATM取现行为。
- 逾期状态。
- 最小还款压力。

## POS_CASH_balance

粒度：历史 POS/cash loan 月度余额。

用途：

- 构建历史现金贷/POS贷款状态。

关键特征方向：

- 剩余期数。
- 合同状态。
- 月度逾期状态。
- 最近几期表现。
